## 1. 과제 정의

- **과제명**: 사내 Serendipity 매칭 AI — 숨은 동료 찾기 커피챗
- **Input**: HR_AX.md(회사 HR AX 방향성·후보 Pool), 차별화 축 결정(의도된 비대칭/Graph distance), 외부 리서치 요약, 첨부1 과제정의서.pdf(흐름 참고용)
- **Output**:
  ① AX 실험실 제출서 (과제유형·과제명·참여자·기대효과 등 간단히)
  ② 1페이지 컨셉 기획서 (As-is / To-be / 활용 Data / 기대효과 흐름)
- **Done 기준**: ①+② 산출, 차별화 축 한 줄 명시, 첫 시범 시나리오 *전사 랜덤(β)*, Phase 2 확장 카드(설명 가능 매칭·시간적 신호) 짧게 표기

## 2. 5단계 매뉴얼

### Clarify
- 참여자 라인업 (혼자 vs 동료 포함) — 작성 시 사용자에게 재확인
- 매칭 거절권·빈도 제한·프라이버시 등 안전장치 명시
- 시범 데이터 소스: One HR DB '26.10 완료 → 그 이전 *부분 데이터* 활용 범위 정의 필요

### Context Gather
- HR_AX.md (프로젝트 루트)
- 첨부1__과제정의서.pdf (제출 불필요 / 1페이지 기획서 뼈대로 활용)
- 외부 리서치 포인터: RandomCoffee.com, Neo4j Talent Graph(DXC·NASA·Gloat), `neo4j-product-examples/genai-workshop`

### Plan
*Claude Code 결정*. 산출물 2건 → 제출서 → 기획서 순 작성 권장.

### Generate
단일 스킬(글쓰기). 회사 톤은 ipc-ppt 스킬 참조 (PPT 아님, 톤만).

### Evaluate
자체 리뷰 → 사용자 검수. 체크: 두 산출물 모순 없음, 차별화 축 일관성, Phase 2 카드 표기.
