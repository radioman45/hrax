가명·가상 사진 사용, 본 시범 범위(전사 랜덤β·의도된 비대칭) 준수.

### 5.1 화면 구성

- **좌측 슬림 앱 바**: 활동/채팅(활성)/팀/일정/통화/파일
- **좌측 채팅 리스트**: `Serendipity Bot` (고정), 매칭 확정 시 상대 채팅방 동적 추가

### 5.3 데모상 시각화하는 차별화 축

- **의도된 비대칭(Graph distance)**: "조직 거리 4단계", "최근 1년 협업 0회" 메트릭을 매칭 사유 카드에 명시 노출
- **사용자 맥락 활용**: 입력 칩과 상대 프로필 칩의 일치 항목을 동적 비교(`sharedInterests`)
- **안전장치**: 카드 하단 푸터에 "격주 1회 · 거절 가능 · 동의 후 공개" 일관 노출, 거절(다음에/이번 주 쉬기) 분기 별도 처리
- **활동 점수 → 인센티브**: 브론즈(0–49, 사내 카페 쿠폰) → 실버(50–99, 도서비 3만원) → 골드(100–199, 문화비 5만원 + 메이커 인터뷰) → 플래티넘(200+, 경영진 라운드테이블)
- **상호 수락 + 비동기 메시지**: "바로 채팅 연결"이 아니라, 상대가 받은편지함에서 메시지를 읽고 수락해야 후속 진행 — 거절·잠수 부담 완화

### 5.4 데모에서 명시적으로 제외 (Phase 2 카드)

- **설명 가능 매칭(1번 축)** 본격 노출 — 데모는 점수 분해 라벨 수준만 살짝, Phase 2에서 SHAP/근거 인용형 확장
- **시간적 신호(3번 축)** — Phase 2에서 "최근 회사 분위기·이벤트 반영" 추가 예정
- **후기 기반 학습 루프** — Phase 2 (만남 후 짧은 평가 → 매칭 가중치 갱신)

## 6. Databricks 구현 메모

### 6.1 데이터 자산 (Unity Catalog · Delta)

| 카탈로그/스키마 | 테이블 | 내용 | 소스 |
|----------------|--------|------|------|
| `hr_ax.serendipity.dim_employee` | 사번·가명·소속·직급·연차 | One HR DB '26.10 본격 연동 / 그 전 부분 데이터 | One HR DB(부분), 인재DB |
| `hr_ax.serendipity.fact_org_graph` | 사번 페어 · 조직 거리(reporting tree hops) · 공통 매니저 거리 | 일배치 산출 | 조직도 스냅샷 |
| `hr_ax.serendipity.fact_collab_history` | 사번 페어 · 1년 협업 횟수(공동 미팅·협업툴·문서 공동 편집) | 주배치 | M365 Graph API, Confluence |
| `hr_ax.serendipity.user_context` | 사번 · 업무·몰입·관심사·취미·학습·선호 동료성향 (자가 입력) | 사용자 폼 제출 | 데모 폼 |
| `hr_ax.serendipity.user_context_emb` | 사번 · 텍스트 필드별 임베딩 벡터 | Foundation Model API 호출 산출 | bge-large-ko / e5 |
| `hr_ax.serendipity.fact_match_event` | 매치ID · 양 사번 · 추천일 · 점수 분해 · 수락여부 · 메시지·일정·실시 여부 | 이벤트 적재 | 앱 백엔드 |
| `hr_ax.serendipity.fact_activity_points` | 사번 · 이벤트유형 · 점수 · 등급 변동 | 이벤트 적재 | 앱 백엔드 |

모든 PII는 가명 키 매핑(`hr_ax.security.pseudo_map`)을 통해 분리 저장, ABAC(Attribute-Based Access Control)로 기획자/운영자 권한 분리.

### 6.2 매칭 알고리즘 파이프라인

1. **임베딩 갱신** (Workflow Job, Foundation Model API): `user_context` 변경분만 증분 임베딩 → `user_context_emb`
2. **후보 풀 생성** (DLT 또는 SQL Warehouse):
   - 조직 거리 ∈ [3, 6] (의도된 비대칭 윈도우)
   - 최근 1년 협업 횟수 ≤ 2회
   - 활동 시간대 겹침 ≥ 2시간/일
3. **유사도 스코어링** (Vector Search Index, `user_context_emb` 위에 구축):
   - 관심사·학습·몰입영역 임베딩 코사인 유사도
   - 너무 유사하면 페널티(반대 비대칭) 적용
4. **종합 점수 계산** (PySpark UDF 또는 SQL):
   - `score = 28*norm(graph_dist) + 22*shared_interests + 18*current_focus + 9*hobby + 10*low_collab + 13*time_overlap` (가중치는 시범 후 베이지안 튜닝)
5. **추천 결과 저장**: `fact_match_event` (status=`pending`)
6. **알림**: Teams 봇 (Lakehouse App에서 Workflow trigger)

### 6.3 서빙 & 앱 레이어

- **백엔드**: Databricks **Lakehouse App** (FastAPI + Databricks SDK) — 추천 조회, 수락/메시지 송신, 일정 슬롯 계산 API
- **프런트**: 본 데모 HTML을 Lakehouse App 정적 자산으로 호스팅 (Phase 1) → Phase 2에서 Teams Adaptive Card 봇 + 사내 포털 임베드로 분기
- **인증**: SSO(SAML/OIDC) → Unity Catalog `current_user()`로 사번 매핑
- **LLM 호출**: Foundation Model API 또는 외부 모델 (AI Gateway 경유, 감사 로그 자동)
  - 매칭 사유 내러티브 생성 (B 톤 문장)
  - 추천 인사말 템플릿 동적 생성 (Phase 1.5)

### 6.4 일정 슬롯 추천 (스텝 8-A)

- M365 Graph `getSchedule` 또는 사내 캘린더 시스템 연동 → Delta 테이블 `dim_calendar_busy` 동기화
- 양측 빈 시간 ∩ 활동 시간대 선호도 → 상위 3개 슬롯 산출
- 장소 추천: 사내 카페 운영시간 테이블(`dim_workplace_cafe`) 조인

### 6.5 활동 점수 & 보상

- `fact_activity_points` 이벤트 누적 → SQL Warehouse 뷰로 분기별 집계
- 등급 산정 Workflow (월 1회) → HR 운영팀 결재 후 사내 복지몰 쿠폰 발급 API 호출
- **컴플라이언스**: 분기말 익명 통계만 인사평가 시스템과 분리 운영, 점수 자체는 평가 반영 X (브리프 제약 일관)

### 6.6 MLOps & 거버넌스

- **MLflow**: 매칭 모델 버전·실험·가중치 관리, 시범 평가 시 A/B 비교
- **Lakehouse Monitoring**: 매칭 수락률·메시지 응답률·일정 진행률·등급별 분포 대시보드
- **Audit**: AI Gateway + Unity Catalog 시스템 테이블로 LLM 호출·민감정보 접근 전수 로그
- **PII**: 임베딩 입력 직전 가명화 한 번 더 적용, 원본 텍스트는 Delta에 별도 격리 스키마 보관
