## 1. 과제 정의

- **과제명**: AI 랜선펫
- **컨셉**: 사용자가 직접 캐릭터를 고르고 이름을 지어 **함께 키우는** 1:1 정서 케어 펫. 대화·체크인을 통해 EXP를 쌓고 5단계로 진화하는 육성 메커니즘으로 자발적 참여를 유도.
- **Input**:
  - 사용자 발화 (자유 대화, 한 줄, 이모지)
  - 정기 체크인 응답 (Adaptive Card 5단계 무드 이모지)
  - 펫의 **프로액티브 발화에 대한 응답** (사용자가 설정한 시간대/빈도에 펫이 먼저 말 걸기)
- **Output**: 3층 구조
  - **사용자(실시간)**: 펫 응답 메시지 + 펫 캐릭터(진화 단계 + 무드 표정) + EXP/레벨 게이지
  - **사용자(주간)**: "이번 주 나의 상태 리포트" — 펫이 작성한 편지, 요일별 무드, 키워드, 본인 전용
  - **HR(월간)**: 익명 집계 — 참여율, 감정 분포, 키워드 TOP10, 응답률 추이 + **실/팀 단위 드릴다운** (k≥10 익명성 가드)
- **Done 기준**: 한 팀(10명 이상) 2주 파일럿 완료, 응답률 50%+, 사용자 만족도(자유응답 기반) 정성 검증, 펫 진화 Lv.3 도달율 30%+

## 2. 핵심 사용자 경험 사양 (데모 검증 완료)

### 2.1 캐릭터 선택 & 명명
- **6종 캐릭터**: 강아지 / 고양이 / 병아리 / 토끼 / 판다 / 여우 (확장 가능)
- 첫 진입 시 캐릭터 선택 + 이름 직접 입력 모달 (애착 강화)
- 헤더 "🔄 캐릭터 변경" 버튼으로 언제든 교체 가능 (육성 레벨은 유지)
- 종마다 5단계 진화 경로의 시각 표현이 다름

### 2.2 육성 / 진화 시스템 (참여 유도 메커니즘)
- **5단계 진화**: Lv.1 알 → Lv.2 베이비 → Lv.3 청소년 → Lv.4 성체 → Lv.5 마스터
  - 종별 진화 이모지 예: 강아지 🥚→🐾→🐶→🐕→🦮 / 고양이 🥚→🐾→🐱→🐈→🦁
- **EXP 누적 임계값**: Lv2=50 / Lv3=150 / Lv4=300 / Lv5=500
- **EXP 획득 규칙**:
  - 이모지 체크인 응답: +15 EXP (긍정 무드 시 +5 보너스)
  - 자유 텍스트 메시지: +5 EXP (5자 이상), +2 EXP (짧은 메시지)
  - 첫 일일 대화 시 연속 출석 보너스: +10 × streak 일수
- **시각적 보상**:
  - 채팅 영역 좌측 상단 sticky 펫 카드 (이모지·이름·Lv 배지·EXP 게이지·통계)
  - EXP 획득 시 "+N EXP" 플로팅 토스트
  - 레벨업 시 상단 토스트 + 펫 회전·반짝 애니메이션 + 채팅 축하 메시지
  - 사이드바·헤더·메시지 아바타가 진화에 맞춰 즉시 변경

### 2.3 일일 무드 표시 (육성과 별개)
- 6단계 무드: 방전 😴 / 다운 🥲 / 평온 😐 / 좋음 🙂 / 활기 😄 / 최고 🤩
- 체크인 응답·자유 메시지 키워드 분석으로 자동 갱신
- 펫 카드의 무드 태그로 표시 (진화 단계와 독립)

### 2.4 펫 프로액티브 메시지 (펫이 먼저 말 걸기)
- **사용자 설정 가능 항목**:
  - 알림 ON/OFF
  - 하루 발화 횟수 (1~6회)
  - 활동 시간대 (시작~종료, 기본 09:00~18:00)
- **시간대 맞춤 메시지 풀**:
  - 아침(06–11): "좋은 아침! 오늘 어떻게 시작했어?"
  - 점심(11–14): "점심 먹었어? 잠깐 한숨 돌리자"
  - 오후(14–17): "오후 집중력 어때? 어깨 한번 펴자"
  - 저녁(17–22): "오늘 고생 많았어! 한 줄로 표현하면?"
  - 야간(22–06): "아직 안 자? 무리하지 말고 쉬어"
  - 30% 확률로 격려성 멘트 (안부, 응원)
- **스케줄링 로직**: 활동 시간을 N등분 후 각 슬롯 중앙 ± 랜덤 jitter로 발화 시점 결정. 오늘 슬롯 종료 시 내일 첫 슬롯으로 이월
- **알림 표시**: 다른 탭/백그라운드 시 사이드바 unread 배지 + 브라우저 탭 타이틀 깜빡임

### 2.5 주간 개인 리포트
- 펫이 작성한 편지 형식 ("이번 주는 화요일에 살짝 다운됐다가...")
- 요일별 무드 그리드
- 자주 등장한 키워드 태그
- **본인만 열람** (HR 접근 차단 명시)

### 2.6 HR 월간 집계 대시보드
- **조회 범위 필터**: 전사 / 실 단위 / 팀 단위 3단계
- KPI 4종: 참여 인원, 주간 응답률, 평균 무드 스코어, 재사용 의향 (전월비 델타 포함)
- 차트: 감정 분포 도넛, 주차별 응답률 추이, 키워드 TOP10
- **부서별 톤 차이 반영**: 실/팀 단위 키워드 풀이 다름 (예: 개발실 "배포/장애/스프린트", 재무실 "결산/마감/보고", HR실 "면담/평가/소통")
- **k-익명성 가드 (k≥10)**: 응답자 10명 미만 단위는 자동 마스킹. 셀렉트 옵션에 🔒 표시, 선택 시 KPI는 잠금 카드, 차트는 흐림 처리
- ⚠ **v1 대비 변경**: "부서간 비교 리포트 제외" → **실/팀 단위 드릴다운 허용 (단, k≥10 강제)**. 노조 협의 시 이 변경점 사전 설명 필요.

## 3. Databricks 구현 매핑

### 3.1 데이터 레이어 (Delta Lake + Unity Catalog)
- `pet_users` — opt-in 사용자, 사번 SSO 매핑 (Unity Catalog row-level security)
- `pet_profiles` — 사용자별 펫 종(species), 이름, 진화 레벨, 누적 EXP, streak, 무드 idx
- `pet_messages` — 대화 로그 (사용자/펫 발화, timestamp, 무드 태그, 키워드 추출 결과). **사용자 본인만 조회 가능** (RLS)
- `pet_checkins` — 체크인 응답 (mood_idx, 시각, 자동 EXP 부여 트리거 결과)
- `pet_proactive_schedule` — 사용자별 프로액티브 발화 설정(빈도/시간대) + 다음 발화 예정 시각
- `org_hierarchy` — 사번 → 팀 → 실 매핑 (HR 집계용)
- `hr_monthly_agg` — 월별 익명 집계 마트 (실/팀 단위 사전 집계, k<10 자동 NULL 마스킹)

### 3.2 AI 레이어 (Mosaic AI)
- **Foundation Model API**: 펫 응답 생성 (시스템 프롬프트로 캐릭터 톤 + 무드 + 진화 단계 반영)
- **AI Functions / `ai_query()`**: 자유 메시지의 무드 분류 + 키워드 추출 (배치 + 실시간)
- **Vector Search**: 시간대·문맥별 프로액티브 메시지 풀 검색 (사용자별 톤 학습 후 개인화)
- **Model Serving Endpoint**: Teams Bot 백엔드가 호출하는 단일 엔드포인트

### 3.3 워크플로우
- **Databricks Workflows (스케줄링)**:
  - 매분 — 프로액티브 발화 큐 처리 (`pet_proactive_schedule.next_visit_at` 도래 사용자 → Teams Bot 호출)
  - 매시 — EXP 누적/레벨업 트리거 재계산 (실시간 처리에서 누락된 케이스 보정)
  - 일일 — streak 갱신, 주간 리포트 사전 집계
  - 월간 — `hr_monthly_agg` 마트 갱신, k-익명성 마스킹 적용

### 3.4 프론트엔드 (Teams)
- **Azure Bot Service** — Teams 개인 채팅 Bot 등록 (백엔드 = Databricks Model Serving Endpoint)
- **Adaptive Card** — 체크인 카드 (5 무드 버튼), 주간 리포트 카드, 캐릭터 선택 카드, 알림 설정 카드
- **펫 시각 자산** — Unity Catalog Volume에 종×진화단계×무드 매트릭스 (현재 데모는 이모지로 대체, 실 운영 시 일러스트 교체)

### 3.5 HR 대시보드
- **Databricks AI/BI Dashboard** — `hr_monthly_agg` 기반, 실/팀 단위 필터 위젯, k<10 자동 마스킹 룰
- 또는 Lakeview Dashboard로 임베드

## 5. 제약사항

- **민감정보**: 대화 원문은 사용자 본인만 열람. HR은 익명 집계만. Unity Catalog row-level security 필수
- **인증**: 사번/비번 로그인 = 사내 SSO 연동 (별도 비번 X)
- **프라이버시 라인**:
  - opt-in 방식, 탈퇴 시 데이터 삭제 보장
  - **k≥10 익명성 강제** (실/팀 단위 드릴다운은 이 가드 하에서만 허용)
  - 부서간 직접 비교 차트는 표시하지 않음 (단일 단위 조회만)
- **프로액티브 발화 가드**:
  - 사용자가 설정한 시간대 외 발화 금지
  - 야간(22~06) 발화는 사용자 명시 동의 시에만
  - 미응답 누적 시 자동 빈도 감소 (스팸 방지)
- **금지**: Graph API 객관 데이터 수집 금지(Phase 2 보류), 개인 식별 가능 형태로 HR 리포트 생성 금지, 노조법/개인정보보호법 검토 미완료 시 전사 확산 금지
