# ADR-002: 에이전트 3분리 구조 (data-analyst / report-writer / qa-reviewer)

**날짜:** 2026-04-18
**상태:** 확정

## 맥락

전체 워크플로우(엑셀 파싱 → 리포트 생성 → 품질 검증)를 하나의 에이전트가 처리할 수도 있고, 역할별로 나눌 수도 있었다. 설계 초기에 몇 가지 분리 방식을 검토했다.

## 결정

역할이 다른 세 단계를 **세 개의 독립 에이전트**로 분리했다.

## 이유

- **각 단계가 사용하는 스킬과 도구가 다르다.** 파싱은 pandas/엑셀 도구, 생성은 python-docx, 검증은 XML 파싱이 중심이다. 하나의 에이전트에 모두 넣으면 컨텍스트가 불필요하게 커진다.
- **배치 생성 시 report-writer만 병렬 복제할 수 있다.** 분리되어 있어야 L002~L010을 9개 독립 인스턴스로 동시 실행할 수 있다. 통합 에이전트라면 이 구조가 불가능하다.
- **QA는 생성 에이전트와 독립되어야 객관성이 보장된다.** 같은 에이전트가 만들고 검수하면 자기 결과물의 문제를 놓칠 위험이 있다.

## 결과

- 에이전트 파일이 3개로 늘어 관리 대상이 증가했다.
- 대신 특정 에이전트만 교체하거나 수정할 수 있어 유지보수가 쉽다. 예를 들어 리포트 디자인을 바꾸려면 report-writer와 feedback-report-writer 스킬만 수정하면 된다.
