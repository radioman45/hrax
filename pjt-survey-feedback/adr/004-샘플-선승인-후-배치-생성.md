# ADR-004: 샘플 1개 선승인 후 나머지 배치 생성

**날짜:** 2026-04-18
**상태:** 확정

## 맥락

10개 리포트를 한 번에 생성할 수도 있었다. 하지만 리포트 형식이 처음 확정되지 않은 상태에서 10개를 한꺼번에 만들면, 형식이 마음에 들지 않을 경우 전체를 다시 만들어야 하는 낭비가 발생한다.

## 결정

L001 샘플을 먼저 생성하고, **사용자가 육안으로 확인·승인한 이후에만** 나머지 9개를 배치 생성한다.

## 이유

- **리포트 형식은 코드로 보장하기 어렵다.** 2페이지 분량, 가독성, 브랜드 느낌은 자동 검증보다 사람의 눈으로 확인하는 것이 더 정확하다.
- **수정 비용을 최소화한다.** 샘플 단계에서 피드백을 반영하면 나머지 9개는 자동으로 올바른 형식으로 생성된다. 10개를 모두 만든 뒤 수정하는 것보다 훨씬 효율적이다.
- 실제로 v1 첫 실행에서 색상 변경 요청이 샘플 승인 전에 발생했고, 이 구조 덕분에 배치 생성 전에 수정이 적용됐다.

## 결과

- 전체 생성 시간이 늘어나는 것처럼 보이지만, 재작업을 방지하므로 실질적인 총 소요 시간은 줄어든다.
- 사용자가 응답하지 않으면 워크플로우가 멈춘다. 자동화 파이프라인에서 사람의 승인을 기다리는 단계가 포함되어 있다는 점을 인식해야 한다.
