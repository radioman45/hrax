# ADR-001: 하이브리드 서브 에이전트 실행 모드 선택

**날짜:** 2026-04-18
**상태:** 확정

## 맥락

리포트 10개를 자동 생성하는 워크플로우를 설계할 때, 에이전트들이 서로 대화하며 협력하는 "팀 모드"와 각자 독립적으로 작업하고 결과만 반환하는 "서브 에이전트 모드" 중 하나를 선택해야 했다.

## 결정

팀 모드 대신 **하이브리드 서브 에이전트 모드**를 선택했다. Phase마다 독립적인 서브 에이전트를 호출하고, 배치 생성(Phase 4)과 QA(Phase 5)는 병렬로 실행한다.

## 이유

- **에이전트 간 실시간 소통이 불필요하다.** 데이터 파싱 → 리포트 생성 → QA는 순서가 고정된 파이프라인이며, 중간에 에이전트끼리 의견을 조율할 일이 없다.
- **리포트 10개는 서로 독립적이다.** L002 리포트가 L003 리포트의 결과에 영향받지 않으므로 병렬 실행이 자연스럽다.
- **팀 모드의 통신 오버헤드가 이득보다 크다.** TeamCreate/SendMessage 설정 비용 대비 실질적인 협업 효과가 없는 구조다.

## 결과

- 배치 생성 9개를 병렬로 실행하여 속도가 크게 향상됐다.
- 반면, 특정 에이전트가 실패했을 때 다른 에이전트가 자동으로 보완하지 못한다. 실패 감지와 재시도는 오케스트레이터가 직접 처리해야 한다.
